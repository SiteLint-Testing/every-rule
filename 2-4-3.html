WCAG 2.4.3: Focus Order

View main list

Brief summary: When a page can be navigated sequentially, focusable components must receive focus in an order that preserves meaning and operability.

View the W3C Understanding page for WCAG 2.4.3 Focus Order.

Observed SiteLint behavior

SiteLint appropriately reports the positive tabindex example and the redundant tabindex="0" example under WCAG 2.4.3. It also uses focus-order-preserved as a manual-verification trigger for focus order.

The example links point to a real page so unrelated same-page-link findings do not obscure WCAG 2.4.3 behavior.

Recommendation

Keep positive-tabindex and misused-tabindex-attribute as concrete partial 2.4.3 checks. Treat focus-order-preserved as a manual-review prompt unless SiteLint adds rendered focus-order analysis that compares the keyboard sequence with visual order, reading order, and workflow order.
