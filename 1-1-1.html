WCAG 1.1.1: Non-text Content

Brief summary: Non-text content that is presented to users needs a text alternative that serves the equivalent purpose, except for specific cases such as decorative content, controls, CAPTCHA, time-based media, tests, and sensory experiences.

View the W3C Understanding page for WCAG 1.1.1 Non-text Content.

The examples below are intended to test common, web-relevant 1.1.1 scenarios that an automated accessibility tool may reasonably evaluate. Each example is labeled as passing or failing and includes a short explanation. Based on this page, SiteLint successfully detects the five intentionally failing examples below and does not appear to flag any of the intended passing examples.

Passing examples

Passing example 1: Informative image with meaningful alternative text

This succeeds because the image conveys information and the alt attribute provides a concise text alternative that communicates the image's purpose.

SiteLint observation: SiteLint does not report this valid informative-image example as an issue, which is appropriate.

Passing example 2: Decorative image ignored by assistive technology

This succeeds because the image is purely decorative and uses alt="", allowing assistive technologies to ignore it.

SiteLint observation: SiteLint does not report this valid decorative-image example as an issue, which is appropriate.

Passing example 3: Image submit button with text alternative describing its purpose

This succeeds because the image-based submit control has alternative text that identifies the action it performs.

SiteLint observation: SiteLint does not report this properly labeled image submit button as a 1.1.1 issue, which is appropriate.

Passing example 4: Image map area with text alternative

This succeeds because the clickable <area> element has an alt attribute that identifies the destination of the mapped region.

SiteLint observation: SiteLint does not report the properly labeled image map areas as 1.1.1 issues, which is appropriate.

Failing examples

Failing example 1: Informative image missing alternative text

This fails because the image conveys information, but it has no alt attribute or other text alternative.

SiteLint observation: SiteLint correctly reports this missing text alternative under its “Missing ALT on non-text elements” rule.

Failing example 2: Image alternative text is a generic placeholder

This fails because the image conveys information, but the alternative text alt="image" does not provide an equivalent description of the content.

SiteLint observation: SiteLint correctly reports this placeholder alternative text under its “Generic placeholder alt text should be avoided” rule.

Failing example 3: Image submit button missing alternative text

This fails because the image-based submit control has no text alternative to identify its purpose.

SiteLint observation: SiteLint correctly reports this image submit button because it lacks alternative text describing its action.

Failing example 4: Image map area missing alternative text

This fails because the clickable <area> element has no alt attribute to identify the purpose of that mapped region.

SiteLint observation: SiteLint correctly reports the unlabeled image map area under its “Image map area must have a text alternative” rule.

Failing example 5: Meaningful SVG with no accessible name

This fails because the meaningful SVG is exposed as an image-like graphic but does not provide an accessible name through <title>, aria-label, or a valid labeling reference.

Current 1.1.1 coverage takeaway

Recommendation: Based on this first-pass 1.1.1 page, SiteLint shows credible automated coverage for several common, machine-detectable non-text content failures: missing text alternatives on informative images, image submit buttons, and image map areas; placeholder alternative text; and meaningful SVGs without accessible names.

These results do not establish complete coverage of every possible 1.1.1 scenario, but they do provide concrete evidence that SiteLint meaningfully detects several important categories of automated 1.1.1 failures.
