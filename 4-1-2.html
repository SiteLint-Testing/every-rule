WCAG 4.1.2: Name, Role, Value

View main list

Brief summary: User interface components must expose their name and role, and states, properties, and values that can be set by the user must be programmatically determinable.

View the W3C Understanding page for WCAG 4.1.2 Name, Role, Value.

Passing examples

Passing example 1: Button has an accessible name

The button exposes a role and name through native HTML text.

Passing example 2: Icon button has an accessible name

The icon is hidden from assistive technologies and the button has an aria-label.

Passing example 3: Iframe has a title

The iframe exposes a name through its title attribute.

Failing examples

Failing example 1: Empty button has no accessible name

The button has no text, label, or referenced label to expose as its name.

Failing example 2: Iframe is missing a title

The iframe has no accessible name.

Failing example 3: Focusable element hidden from assistive technologies

The focusable button is inside content marked aria-hidden="true", so keyboard users may reach something that assistive technologies cannot expose.

Failing example 4: Custom switch does not expose role or state

This visually resembles a switch, but it does not expose a switch role or checked state.
