WCAG 1.4.6: Contrast (Enhanced)

View main list

Brief summary: WCAG 1.4.6 is the AAA enhanced contrast requirement. Normal text generally requires at least 7:1 contrast, while large-scale text generally requires at least 4.5:1 contrast.

View the W3C Understanding page for WCAG 1.4.6 Contrast (Enhanced).

These examples focus on machine-detectable text contrast cases. This page does not attempt to test contrast inside images of text, because that requires image analysis or human review.

Relationship to WCAG 1.4.3

WCAG 1.4.6 uses stricter contrast thresholds than WCAG 1.4.3. Any text that fails WCAG 1.4.3 AA will also fail WCAG 1.4.6 AAA. More importantly for this page, some examples that pass WCAG 1.4.3 should still fail WCAG 1.4.6 because they meet the AA threshold but not the AAA threshold.

Observed missed example 6: Placeholder text fails enhanced contrast

This is a meaningful 1.4.6 failure because placeholder text can provide instructions or examples and should meet the applicable enhanced contrast threshold when it conveys information. During testing, SiteLint did not report this placeholder contrast failure.

Observed result: SiteLint did not flag the low-contrast placeholder text. This should be treated as a rule-expansion area for enhanced contrast as well.

Recommendation: Add placeholder contrast support if SiteLint intends to evaluate all text-like content that can convey instructions or examples. Placeholder text should be evaluated when it communicates information to users.

Observed missed example 7: Text over a gradient background may fail enhanced contrast

This is a meaningful 1.4.6 risk because the contrast depends on the rendered pixels behind each part of the text. A CSS-only contrast check may not reliably evaluate gradients, images, video backgrounds, or patterned backgrounds.

Observed result: SiteLint did not report this gradient-background example. The actual contrast depends on the rendered background behind the text, so this should be treated as a rendered-analysis or rule-expansion area.

Recommendation: Consider rendered-background analysis for text over gradients, images, video, or patterned backgrounds. This likely requires screenshot or pixel-based testing rather than CSS-only evaluation.

Observed missed example 8: Parent opacity reduces rendered text contrast

This is a meaningful 1.4.6 risk because the computed text color may appear sufficient, while the rendered text is visually lighter because opacity is applied to an ancestor element.

This text may appear lower contrast because its parent has opacity.

Observed result: SiteLint did not report this opacity example. A rule that reads only the computed color and background-color may overestimate contrast if it does not account for ancestor opacity, alpha colors, overlays, or compositing effects.

Recommendation: Account for opacity on the text node and ancestor elements when calculating contrast. If SiteLint cannot compute the final composited colors reliably, this should be reported as a manual-review candidate rather than a confirmed pass.

Observed missed example 9: SVG text fails enhanced contrast

This is a meaningful 1.4.6 risk because SVG <text> can be visible, meaningful text content. HTML-only text contrast checks may miss SVG text entirely.

Observed result: SiteLint did not report this SVG text example. This should be treated as a rule-expansion area if SiteLint intends to evaluate meaningful visible SVG <text> under WCAG 1.4.6.

Recommendation: Evaluate SVG <text> contrast when SVG text is visible and meaningful. Complex SVG cases involving masks, filters, transforms, layering, or patterned backgrounds may still require rendered analysis or manual review.

Observed SiteLint coverage notes

SiteLint reports the core enhanced-contrast failures that distinguish WCAG 1.4.6 from WCAG 1.4.3. These include examples that pass WCAG 1.4.3 AA but fail the stricter WCAG 1.4.6 AAA thresholds.

SiteLint incorrectly reports the decorative aria-hidden low-contrast divider. SiteLint also did not report the placeholder text, gradient background, opacity/compositing, or SVG text examples. These should be treated as rule-refinement or rule-expansion areas.

Recommended SiteLint rule improvements

Recommendation: Keep WCAG 1.4.6 as a separate enhanced-contrast rule from WCAG 1.4.3. It should use the stricter 7:1 threshold for normal text and 4.5:1 threshold for large-scale text.

Recommendation: Reuse the same rule improvements recommended for 1.4.3: avoid confirmed failures for purely decorative aria-hidden text, add placeholder contrast support, and consider rendered-background, opacity/compositing, and SVG text contrast enhancements.

Recommendation: Based on the observed results for this page, specifically prioritize four gaps: decorative aria-hidden false positives, placeholder text contrast, rendered-background analysis, and SVG <text> contrast. Opacity and compositing should also be evaluated as part of rendered contrast analysis.
