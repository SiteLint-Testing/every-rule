WCAG 1.2.3: Audio Description or Media Alternative (Prerecorded)

Brief summary: Prerecorded synchronized media with video content needs either an audio description of important visual information or a complete alternative for time-based media that presents the essential visual and auditory information in text form. This requirement does not apply when the media is clearly labeled as a media alternative for text.

View the W3C Understanding page for WCAG 1.2.3 Audio Description or Media Alternative (Prerecorded).

These examples focus on structural cases an automated accessibility tool may be able to evaluate. Whether an audio description or text alternative fully conveys the necessary visual information remains a human judgment.

Passing examples

Passing example 1: Prerecorded synchronized media with an adjacent link to a full media alternative

This succeeds because a link immediately following the video points to a detailed alternative for time-based media that provides the essential visual and auditory information in text form.

SiteLint observation: SiteLint currently reports this passing example as a WCAG 1.2.1 issue because the video does not contain a <track> element or audio source. That is not a valid basis for failing this 1.2.3 example. WCAG 1.2.3 allows a full media alternative in text form, and this scenario provides one.

Recommendation: SiteLint needs explicit WCAG 1.2.3 coverage that recognizes both valid passing paths: audio description and a complete media alternative for time-based media. A generic “no track or audio source” finding should not be used as a confirmed failure when a full text alternative is present.

Passing example 2: Media clearly labeled as an alternative for text

This succeeds under the 1.2.3 exception because the video is explicitly identified as a media alternative for the adjacent text. Additional audio description or a separate media alternative is not required when synchronized media is a clearly labeled media alternative for text.

SiteLint observation: SiteLint currently reports this passing example as a WCAG 1.2.1 issue, even though WCAG 1.2.3 does not require audio description or a separate media alternative when the synchronized media is itself clearly labeled as a media alternative for text.

Recommendation: WCAG 1.2.3 coverage should account for the “media alternative for text” exception. If SiteLint cannot reliably confirm this exception automatically, it should not treat the absence of a <track> element or audio source as a confirmed failure.

Failing examples

Failing example 1: Prerecorded synchronized media with no audio description or media alternative

This fails because the video includes important visual information, but the page provides neither audio description nor a complete alternative for time-based media.

SiteLint observation: SiteLint flags this genuinely failing scenario, but it reports the issue under WCAG 1.2.1 rather than WCAG 1.2.3. The result is directionally useful, but it does not demonstrate accurate 1.2.3 coverage.

Recommendation: SiteLint should have explicit WCAG 1.2.3 reporting for prerecorded synchronized media that lacks both audio description and a media alternative, whether through a dedicated rule or a broader media rule with correct SC mapping.

Current 1.2.3 coverage takeaway

Recommendation: SiteLint currently appears to use the same broad media rule across the 1.2.1, 1.2.2, and 1.2.3 test pages, but it reports these findings only as WCAG 1.2.1. See 1.2.1, 1.2.2, and this 1.2.3 page for the related scenarios.

That approach creates both mapping and logic problems: the rule incorrectly flags valid 1.2.2 and 1.2.3 passing scenarios, misses at least one 1.2.2 failure scenario, and uses a single 1.2.1 label for distinct success criteria with different requirements. SiteLint needs clearer, accurately mapped media coverage across these SCs.
