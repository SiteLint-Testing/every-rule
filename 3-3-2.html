Failing examples

Failing example 1: Input has no label

The text input requires user input, but there is no visible or programmatic label.

Failing example 2: Placeholder is used as the only instruction

The field requires a specific date format, but the only instruction is inside the placeholder. Placeholder text can disappear while the user is entering data and is not a reliable replacement for a persistent label or instruction.

Failing example 3: Required field is not explained

The asterisk visually marks the field as required, but the page does not explain what the asterisk means.

Failing example 4: Fieldset has an empty legend

The fieldset's first child is a legend, but the legend has no text. This does not provide a usable label or instruction for the grouped controls.

Failing example 5: Fieldset legend is not the first child

The fieldset has a legend, but another element appears before it. The legend should be the first child so the grouped controls have a reliable caption.

Observed SiteLint behavior

SiteLint's active 3.3.2 finding on this page comes from legend-first-child-of-fieldset. That rule evaluates only fieldset elements by checking whether the first child is a non-empty legend.

The rule appropriately covers the grouped-control legend examples, including an empty legend and a legend that is not the first child. However, the non-legend-first-child case also overlaps with the separate 1.3.1 fieldset-no-legend rule, so that example may produce more than one related finding.

SiteLint does not currently evaluate several important 3.3.2 scenarios on this page through the 3.3.2 rule: an ordinary input with no label, a required format provided only as placeholder text, unexplained required-field indicators, or whether instructions are sufficiently clear for the task.

Recommended SiteLint treatment

Recommendation: Keep legend-first-child-of-fieldset as useful automated coverage for grouped controls, but do not present it as complete WCAG 3.3.2 Labels or Instructions coverage.

To evaluate this page more effectively, SiteLint should add or map rules for unlabeled form controls, placeholder-only labels or instructions, missing required-field explanations, missing format instructions, and helper text that is not programmatically associated with the relevant field.

Some instruction-quality issues should remain manual-review candidates. The tool can identify likely risks, but deciding whether an instruction is complete, understandable, and appropriate for the task may require human judgment.
