WCAG 2.4.7: Focus Visible

View main list

Brief summary: Keyboard focus indicators must be visible when a component receives keyboard focus.

View the W3C Understanding page for WCAG 2.4.7 Focus Visible.

Observed SiteLint behavior

SiteLint uses focus-visible as a manual-verification trigger for WCAG 2.4.7. This is useful as a reminder to inspect focus indicators, but it does not appear to evaluate whether the focused controls actually pass or fail.

The example links point to a real page and the review input has a label so unrelated same-page-link and form-label findings do not obscure visible-focus behavior.

Recommendation

Keep focus-visible as a manual prompt unless SiteLint adds rendered focus-state testing. Stronger coverage should move focus through each interactive component and identify removed, transparent, or visually imperceptible focus indicators.
