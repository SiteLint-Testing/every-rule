WCAG 2.5.8: Target Size (Minimum)

View main list

Brief summary: Pointer targets should be at least 24 by 24 CSS pixels, or have enough spacing so a 24 CSS pixel diameter circle centered on the target does not intersect another target or the circle for another undersized target. Exceptions may apply.

View the W3C Understanding page for WCAG 2.5.8 Target Size (Minimum).

What this page is intended to test

This page tests whether SiteLint evaluates WCAG 2.5.8 Target Size (Minimum), which is different from WCAG 2.5.5 Target Size (Enhanced). WCAG 2.5.8 uses a 24 by 24 CSS pixel minimum target-size model and allows spacing-based alternatives and exceptions.

A useful automated or semi-automated rule should identify the specific undersized target, measure its rendered dimensions, evaluate nearby target spacing where possible, and account for exceptions such as inline text links, equivalent controls, user-agent-controlled targets, and essential small targets.

Observed SiteLint result

SiteLint reported nine WCAG 2.5.8 findings: “Interactive elements must have an area of at least 24 by 24 pixels.” The findings identify specific undersized controls and provide element paths. This indicates that SiteLint is performing element-level target-size detection for WCAG 2.5.8, not merely showing a generic manual-verification prompt.

SiteLint correctly reported the intentionally undersized standalone link, standalone button, crowded adjacent buttons, and spaced undersized buttons. The reported dimensions were 18 by 18 CSS pixels for most button examples and 20 by 20 CSS pixels for the small standalone link.

SiteLint also reported the small button in passing example 4, even though that example includes an equivalent larger “Download report” button next to it. This suggests the rule measures undersized targets but does not fully evaluate the WCAG 2.5.8 equivalent-target exception.

Recommended SiteLint treatment

Recommendation: Distinguish WCAG 2.5.8 Target Size (Minimum) from WCAG 2.5.5 Target Size (Enhanced). WCAG 2.5.8 uses a 24 by 24 CSS pixel model with spacing allowances, while WCAG 2.5.5 uses a stricter 44 by 44 CSS pixel threshold.

Recommendation: Preserve the current element-level reporting behavior. SiteLint successfully identifies specific undersized targets, reports measured dimensions, provides element paths, and offers highlighting. Improve the rule by adding exception handling for equivalent larger targets and by evaluating nearby spacing where possible.

Recommendation: Account for WCAG 2.5.8 exceptions before presenting a finding as a confirmed failure. Inline text links, equivalent larger targets, user-agent-controlled targets, and essential small targets should be treated carefully. Current testing shows that equivalent larger targets are not fully evaluated. Where SiteLint cannot determine whether an exception applies, report the specific target as a review candidate rather than a confirmed failure.
