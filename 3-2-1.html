WCAG 3.2.1: On Focus

View main list

Brief summary: Receiving focus must not automatically initiate a change of context.

View the W3C Understanding page for WCAG 3.2.1 On Focus.

Observed SiteLint behavior

SiteLint addresses WCAG 3.2.1 with on-focus, which appears to be a broad manual-verification trigger. It does not appear to evaluate whether the examples actually navigate or submit when focus is received.

The focus-navigation example points to a real page so unrelated same-page-link findings do not obscure on-focus behavior.

Recommendation

Keep on-focus as a manual prompt unless SiteLint can inspect or execute focus event behavior. Improved 3.2.1 coverage should detect high-confidence patterns such as focus handlers that navigate, submit forms, open windows, move focus, or substantially replace content.
