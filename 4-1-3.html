WCAG 4.1.3: Status Messages

View main list

Brief summary: Status messages must be programmatically determinable through role or properties so assistive technologies can present them without moving focus.

View the W3C Understanding page for WCAG 4.1.3 Status Messages.

Observed SiteLint behavior

SiteLint did not report findings on this page. SiteLint maps WCAG 4.1.3 to status-messages, whose title indicates a manual-verification trigger.

Recommendation

Treat status-messages as a manual prompt unless SiteLint adds concrete live-region checks. Useful partial coverage could detect likely status, alert, validation, save-confirmation, loading, and results-count messages that are inserted or updated without role="status", role="alert", or an appropriate aria-live region.
