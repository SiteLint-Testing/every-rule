WCAG Test Page Index

This index tracks the WCAG success-criterion test pages being created to evaluate SiteLint coverage. Each linked page contains representative passing and failing scenarios for that success criterion, followed by observations and recommendations when SiteLint behavior has been reviewed.

Created and skipped WCAG test pages
SC Title Notes
1.1.1 Non-text Content
  • No SiteLint issues requiring follow-up identified from the current first-pass test page.
1.2.1 Audio-only and Video-only (Prerecorded)
  • SiteLint incorrectly flags a passing video-only example that provides an associated text alternative, even though WCAG 1.2.1 allows prerecorded video-only content to pass through an alternative for time-based media.
  • The current media rule appears to treat the absence of a track element or audio source as sufficient evidence of failure, which is too narrow for 1.2.1.
  • Recommendation: 1.2.1 logic should account for the valid text-alternative pathway rather than treating the absence of embedded track/audio markup as conclusive.
1.2.2 Captions (Prerecorded)
  • SiteLint incorrectly flags a passing example that is clearly labeled as a media alternative for text, which is an exception under WCAG 1.2.2.
  • SiteLint catches the missing-captions failing scenario only through a finding mapped to 1.2.1, not 1.2.2.
  • SiteLint misses the subtitles-only failing scenario, suggesting that subtitles may be treated as equivalent to captions.
  • Recommendation: SiteLint needs explicit, accurately mapped 1.2.2 coverage and should not treat kind="subtitles" as a guaranteed substitute for captions.
1.2.3 Audio Description or Media Alternative (Prerecorded)
  • SiteLint incorrectly flags a passing example that provides a full media alternative in text form.
  • SiteLint incorrectly flags a passing example that is clearly labeled as a media alternative for text.
  • SiteLint flags the genuine failing scenario, but reports it under WCAG 1.2.1 rather than 1.2.3.
  • Recommendation: SiteLint needs explicit 1.2.3 coverage that accounts for audio description, full media alternatives, and the media-alternative-for-text exception.
1.2.4 Captions (Live)
  • Skipped: This success criterion concerns live captions for live synchronized media.
  • Not a good fit for a static HTML test-page corpus.
1.2.5 Audio Description (Prerecorded)
  • SiteLint incorrectly flags a passing example where important visual information is already conveyed in the main audio track, a valid 1.2.5 scenario.
  • SiteLint flags the genuine failing scenario, but again reports it under WCAG 1.2.1 rather than 1.2.5.
  • Recommendation: SiteLint should not treat the absence of a descriptions track as a confirmed 1.2.5 failure. This often requires focused manual verification.
1.2.6 Sign Language (Prerecorded)
  • Skipped: This success criterion concerns sign language interpretation for prerecorded synchronized media.
  • Not a good fit for static HTML test pages or reliable markup-only automated evaluation.
