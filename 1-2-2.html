WCAG 1.2.2: Captions (Prerecorded)

Brief summary: Prerecorded synchronized media with audio, such as a video that includes speech or meaningful sounds, needs captions that provide equivalent access to the audio information. This requirement does not apply when the media is clearly labeled as a media alternative for text.

View the W3C Understanding page for WCAG 1.2.2 Captions (Prerecorded).

The examples below focus on structural cases an automated accessibility tool may reasonably evaluate: whether prerecorded video with audio includes an associated captions track. Whether the captions are accurate, complete, synchronized, and include important sounds remains a human judgment.

Passing examples

Passing example 1: Prerecorded video with a captions track

This succeeds because the prerecorded video includes a <track kind="captions"> element that provides a captions file for the audio content.

Passing example 2: Prerecorded video clearly labeled as a media alternative for text

This succeeds under the 1.2.2 exception because the video is explicitly identified as a media alternative for the adjacent text content. A separate captions track is not required when synchronized media is a clearly labeled media alternative for text.

SiteLint observation: SiteLint currently reports this passing example as a media accessibility issue, even though WCAG 1.2.2 does not require captions when synchronized media is clearly labeled as a media alternative for text.

Recommendation: 1.2.2 coverage should account for the “media alternative for text” exception. If SiteLint cannot reliably confirm that exception automatically, this should not be treated as a confirmed failure solely because a captions track is absent.

Failing examples

Failing example 1: Prerecorded video with audio and no captions track

This fails because the prerecorded video includes audio content but does not provide captions for that audio information.

SiteLint observation: SiteLint correctly flags this media element as lacking an expected accessibility alternative. However, the current report is mapped to WCAG 1.2.1 rather than WCAG 1.2.2, so it does not yet demonstrate direct 1.2.2 coverage.

Recommendation: SiteLint should have explicit 1.2.2 coverage for prerecorded synchronized media that lacks captions, whether through a dedicated rule or a revised media rule with accurate SC mapping.

Failing example 2: Prerecorded video with subtitles instead of captions

This fails as a 1.2.2 test case because a subtitles track is not the same as a captions track. Captions are intended to provide access to speech and relevant non-speech audio information, while subtitles may only translate dialogue.

SiteLint observation: SiteLint does not report this failing example. That suggests it is treating <track kind="subtitles"> as sufficient, even though subtitles are not automatically equivalent to captions for WCAG 1.2.2.

Recommendation: 1.2.2 coverage should not treat kind="subtitles" as a guaranteed pass. A captions-specific check should distinguish kind="captions" from subtitles, or at minimum avoid treating subtitles alone as conclusive evidence of compliance.

Current 1.2.2 coverage takeaway

Recommendation: SiteLint currently appears to have incomplete or misaligned coverage for WCAG 1.2.2. It flags a valid media-alternative-for-text exception, catches a missing-caption scenario only through a rule mapped to 1.2.1, and misses a subtitles-only scenario that should not be treated as equivalent to captions. SiteLint needs explicit, accurately mapped 1.2.2 coverage.
