WCAG 1.2.1: Audio-only and Video-only (Prerecorded)

View main list

Brief summary: Prerecorded audio-only content needs an alternative for time-based media that presents equivalent information. Prerecorded video-only content needs either an alternative for time-based media or an audio track that presents equivalent information.

View the W3C Understanding page for WCAG 1.2.1 Audio-only and Video-only (Prerecorded).

These examples test whether SiteLint distinguishes valid and invalid implementations of WCAG 1.2.1. For audio-only media, the requirement is an alternative for time-based media that presents equivalent information. For video-only media, the requirement can be met through either an alternative for time-based media or an audio track that presents equivalent information. Whether a provided alternative is fully equivalent remains a human judgment.

Passing examples

Passing example 1: Audio-only media with an adjacent transcript

This succeeds because the audio-only media is followed by a transcript that presents the information conveyed in the recording.

SiteLint observation: This is a valid 1.2.1 passing pattern. If SiteLint reports this as a failure solely because the <audio> element does not contain a <track> element, that would be too narrow; WCAG 1.2.1 permits an equivalent text alternative for prerecorded audio-only content.

Passing example 2: Video-only media with an adjacent text alternative

This succeeds because the silent video-only media is followed by a text alternative that describes the important visual information presented in the video.

SiteLint observation: SiteLint currently reports this passing example as a 1.2.1 issue because the <video> does not include a <track> element or audio source. That appears overly narrow. WCAG 1.2.1 allows prerecorded video-only media to pass through either an equivalent text alternative or an equivalent audio track; this example demonstrates the text-alternative pathway.

Recommendation: The rule should not treat the absence of <track> or an audio source as sufficient evidence of a 1.2.1 failure when an associated text alternative is present. At minimum, this condition should not be reported as a confirmed failure without accounting for the text-alternative option.

Failing examples

Failing example 1: Audio-only media with no transcript or equivalent alternative

This fails because the audio-only media is presented without a transcript or other equivalent alternative for people who cannot hear the recording.

SiteLint observation: This is an appropriate 1.2.1 failure case for SiteLint to report, provided the tool is accurately identifying prerecorded audio-only media with no equivalent alternative.

Failing example 2: Video-only media with no text alternative or equivalent audio track

This fails because the silent video-only media is presented without a text alternative or audio track that explains the important visual information.

SiteLint observation: This is an appropriate 1.2.1 failure case for SiteLint to report. Unlike the passing video-only example above, this media has neither an equivalent text alternative nor an equivalent audio track.

Recommendation: A 1.2.1 rule should distinguish audio-only media from video-only media and should not conflate 1.2.1 with captions or audio-description requirements that belong to other time-based media success criteria.
