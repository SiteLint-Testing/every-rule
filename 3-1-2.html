WCAG 3.1.2: Language of Parts

View main list

Brief summary: The human language of each passage or phrase in the content must be programmatically determinable when it differs from the page's default language.

View the W3C Understanding page for WCAG 3.1.2 Language of Parts.

Observed SiteLint behavior

SiteLint reports the two elements on this page that already have a lang attribute: the Spanish phrase with lang="es" and the French quotation with lang="fr". These findings are marked as manual verification items that ask the reviewer to confirm the element content accurately relates to the declared language.

This matches the current language-of-parts rule implementation, which selects :not(html) [lang]. The rule does not detect unmarked foreign-language phrases. It also does not automatically determine whether the declared language value matches the element content.

Passing examples

Passing example 1: Spanish phrase marked with lang

The Spanish phrase has its own language attribute.

The confirmation message says su pago fue recibido.

Passing example 2: French quotation marked with lang

The quoted French sentence is identified as French.

Le service est disponible pendant les heures ouvrables.

Failing examples

Failing example 1: Spanish phrase not marked

The Spanish phrase differs from the page language but is not marked with a language attribute.

The account page displays the message: su pago fue recibido.

Failing example 2: French quotation not marked

The quotation is in French, but no element identifies that language change.

Le service est disponible pendant les heures ouvrables.

Manual review examples

Manual example 1: Proper names and borrowed words

Some words may look like a language change but do not require a language attribute when they are names, common borrowed terms, or part of the surrounding language.

The report mentions Cafe Central, deja vu, and the El Camino project.

Manual example 2: Mixed-language content needs judgment

A page with many short phrases in several languages may require human review to decide which phrases are meaningful language changes.

The welcome banner includes hello, hola, bonjour, and hallo as decorative examples.

Recommended SiteLint treatment

Recommendation: Treat the current language-of-parts rule as partial manual-verification coverage only. It is useful as a prompt to review declared language changes, but it should not be presented as automated 3.1.2 pass/fail coverage.

If SiteLint expands this rule, separate the two concerns: detecting likely unmarked language changes, and validating the accuracy of existing lang values. Both should remain review-oriented unless the tool has high confidence.
