WCAG 2.5.7: Dragging Movements

View main list

Brief summary: Functionality that uses dragging must also be operable with a single pointer without dragging, unless dragging is essential.

View the W3C Understanding page for WCAG 2.5.7 Dragging Movements.

Observed SiteLint behavior

SiteLint did not report findings on this page. SiteLint maps WCAG 2.5.7 to dragging-movements, whose title indicates a manual-verification trigger.

Recommendation

Treat dragging-movements as a manual prompt unless SiteLint adds concrete detection for drag-dependent controls. Useful partial coverage could flag draggable elements, slider-like widgets, sortable regions, and canvas/map interactions that do not expose an obvious non-drag alternative.
