Potential false positive: decorative linked image with descriptive link text

WCAG Success Criteria relevant: 1.1.1 Non-text Content and 2.4.4 Link Purpose (In Context)

This page tests whether SiteLint treats an image inside a link as needing its own alternative text even when the image is decorative and the link's visible text already provides the full purpose of the link.

Scenario: decorative icon inside a descriptive link

This should be accessible. The image is decorative, so it correctly uses alt="". The adjacent visible link text, Download annual report, provides the link's accessible name and fully communicates the destination.

Download annual report

This text provides a valid same-page destination for the demonstration link.

If SiteLint reports the empty alt attribute as a problem here, that would be a useful example of a rule being too literal: the image does not need its own text alternative because it is decorative, and the link purpose is already conveyed by visible text.
