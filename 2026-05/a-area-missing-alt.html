ARIA Attribute Conflict Test

WCAG Success Criterion tested: 4.1.2 Name, Role, Value

This success criterion requires user interface components to expose accurate programmatic information, including their name, role, and state. Conflicting ARIA attributes can provide contradictory state information to assistive technologies.

Passing example

This succeeds because the button uses only aria-pressed to communicate its toggle state, without adding a conflicting ARIA state.

Failing example

This fails because the button uses both aria-pressed and aria-selected, which can create conflicting or inappropriate state information on the same control.
