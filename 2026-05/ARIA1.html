ARIA1: Using aria-describedby to provide descriptive information for user interface controls

Technique: ARIA1. This technique demonstrates how to use aria-describedby to associate a user interface control with additional descriptive information in the same document.

Related WCAG Success Criteria: 1.3.1 Info and Relationships (Advisory), and 3.3.2 Labels or Instructions (Sufficient when used with G131: Providing descriptive labels).

View the W3C ARIA1 technique.

Passing example

This succeeds because the button uses aria-describedby to reference existing descriptive text that provides additional information about the control.

Failing example 1: Referenced description ID does not exist

This fails because the button uses aria-describedby, but the referenced description ID does not exist in the page. As a result, the intended descriptive information is not programmatically associated with the control.

Failing example 2: aria-describedby is empty

This fails because the button has an aria-describedby attribute, but the attribute is empty. It does not reference any descriptive text.

Failing example 3: Referenced description exists but is empty

This fails because the button references an existing description element, but that element contains no descriptive information about the control.

Failing example 4: Referenced text exists but does not provide useful additional information

This fails the ARIA1 technique in substance because the referenced text does not provide meaningful additional information about the control. This may require human judgment rather than automated detection.
