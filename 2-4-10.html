WCAG 2.4.10: Section Headings

View main list

Purpose of this test: This page intentionally avoids all native heading elements so it can trigger SiteLint's no_headings rule. The usual test-page template is not used here because the template includes headings, and even one native heading prevents this rule from firing.

What this page is doing

The no_headings rule selects only native heading elements: h1, h2, h3, h4, h5, and h6. If the page contains any of those elements, the rule returns without reporting. To trigger the rule, this page uses visually styled div elements for its title and section labels instead of semantic heading elements.

This is a deliberate test-page exception. It is not a recommended authoring pattern. The visual labels are present only so the page remains readable while still containing zero native heading elements.

Failing example 1: Page has no native heading elements

This page contains visually prominent section labels, but none of them use h1 through h6. That should trigger SiteLint's no_headings rule because the rule only checks whether native heading elements exist anywhere in the document.

Expected SiteLint behavior

SiteLint should report the no_headings rule because this document contains no native heading elements. The finding is expected to be page-level because the rule reports with node: null; there is no specific heading element to highlight.

If SiteLint reports this under WCAG 2.4.10, that confirms the rule can be triggered only by removing all native headings from the page. If SiteLint reports no issue, then the rule may not be enabled in the current auditor profile or may not be mapped to the active WCAG filter selection.

Recommended SiteLint treatment

Recommendation: Treat no_headings as a review candidate, not a confirmed WCAG 2.4.10 failure. A page with no native headings may indicate poor section organization, but WCAG 2.4.10 requires judgment about whether the page has sections that need headings and whether those headings organize the content effectively.

Recommendation: Do not treat no_headings alone as complete WCAG 2.4.10 coverage. The rule only detects the absence of native heading elements. It does not evaluate whether section headings are needed, whether existing headings are useful, whether visual section labels should be semantic headings, whether heading text is descriptive, or whether the content is organized effectively.

Recommendation: If SiteLint intends to claim broader WCAG 2.4.10 coverage, it should add explicit review candidates for long pages without meaningful section headings, visually styled headings that are not semantic, repeated generic section headings, and content groups that appear to need headings but lack them.
