Accessible SVG rule test

WCAG Success Criterion tested: 1.1.1 Non-text Content

This rule checks whether a meaningful inline <svg> element has accessible text that identifies it for assistive technology users.

Passing example

This succeeds because the SVG includes a non-empty <title> element that provides an accessible name.

Failing example

This fails because the SVG does not include an accessible name or description, such as a <title>, <desc>, or aria-label.
