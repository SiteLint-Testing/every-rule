C15: Using CSS to change the presentation of a user interface component when it receives focus

Technique: C15. This technique demonstrates using CSS to provide visible feedback when an interactive element receives keyboard focus.

Related WCAG Success Criteria: 1.4.1 Use of Color (Advisory) and 2.4.7 Focus Visible (Sufficient).

View the W3C C15 technique.

Passing example

This succeeds because the links visibly change presentation when they receive keyboard focus. Tabbing to a link applies a background color, text color, and visible outline through :focus-visible.

Failing example 1: Visual styling appears on hover, but not on keyboard focus

This fails because the links change presentation when hovered with a mouse, but they do not receive an equivalent visible CSS focus treatment when reached with the keyboard. The default outline has also been removed, so keyboard users do not get a visible focus indicator.

Failing example 2: Focus styling is removed without replacement

This fails because the links remove the browser's default focus outline and do not replace it with any visible focus treatment. Tabbing to the links does not produce a visible change in presentation.
