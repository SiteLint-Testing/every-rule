WCAG 1.2.5: Audio Description (Prerecorded)

Brief summary: Prerecorded synchronized media with video content must provide audio description for important visual information that is not already conveyed through the existing audio track. If all important visual information is already communicated in the original audio, additional audio description is not required.

View the W3C Understanding page for WCAG 1.2.5 Audio Description (Prerecorded).

These examples focus on structural scenarios that may help reveal whether an automated tool is directionally aligned with WCAG 1.2.5. Whether a provided audio description is accurate, complete, and synchronized remains a human judgment.

Passing examples

Passing example 1: Prerecorded synchronized media with an audio-description track

This succeeds because the video includes a <track kind="descriptions"> element intended to provide audio description for important visual content.

Passing example 2: Important visual information is already conveyed in the main audio track

This succeeds because WCAG 1.2.5 does not require additional audio description when all important visual information is already conveyed through the video's existing audio. This is a valid exception, although an automated tool may not be able to verify it from markup alone.

SiteLint observation: SiteLint currently reports this passing example as a WCAG 1.2.1 issue because the video does not include a <track> element or audio source. That is not a valid basis for failing this 1.2.5 scenario. WCAG 1.2.5 does not require separate audio description when the existing audio already conveys all important visual information.

Recommendation: SiteLint should not treat the absence of <track kind="descriptions"> as a confirmed 1.2.5 failure. At most, this may justify targeted manual verification for prerecorded synchronized media, because the need for audio description depends on the relationship between the visuals and the existing audio.

Failing examples

Failing example 1: Prerecorded synchronized media with important visual information and no audio description

This fails because the video includes important visual information that is not conveyed through the audio track, but no audio description is provided.

SiteLint observation: SiteLint flags this genuinely failing scenario, but it reports the issue under WCAG 1.2.1 rather than WCAG 1.2.5. The result is directionally useful, but it does not demonstrate accurate 1.2.5 coverage.

Recommendation: SiteLint needs explicit WCAG 1.2.5 coverage for prerecorded synchronized media where audio description may be required. Because the failure depends on media content, the most defensible automated result may often be a focused manual-verification prompt rather than a confirmed failure.

Current 1.2.5 coverage takeaway

Recommendation: SiteLint appears to apply the same broad media rule used on the 1.2.1, 1.2.2, and 1.2.3 pages, and it again reports results only as WCAG 1.2.1. On this 1.2.5 page, that causes a false positive on a valid passing scenario and a mis-mapped result on a genuine failing scenario.

For 1.2.5 specifically, the presence of a descriptions track can be useful positive evidence, but the absence of one is not enough to confirm failure. SiteLint needs more precise media-rule logic and more accurate WCAG success-criterion mapping.
