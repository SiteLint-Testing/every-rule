WCAG 1.2.8: Media Alternative (Prerecorded)

View main list

Brief summary: A full alternative for time-based media must be provided for all prerecorded synchronized media and all prerecorded video-only media. This alternative presents the essential visual and auditory information in text form and is more comprehensive than captions or ordinary audio description alone.

View the W3C Understanding page for WCAG 1.2.8 Media Alternative (Prerecorded).

Caution: This page can test whether a clearly associated media alternative is present or absent, but it cannot verify whether that alternative fully captures all visual and auditory information in the media. A real conformance decision requires human review of both the media and the alternative.

Failing examples

Failing example 1: Prerecorded synchronized media with no media alternative

This fails because the page presents prerecorded synchronized media but provides no full alternative for time-based media.

SiteLint observation: SiteLint flags this genuinely failing scenario, but it reports the issue under WCAG 1.2.1 rather than WCAG 1.2.8. The result is directionally useful, but it does not demonstrate accurate 1.2.8 coverage.

Recommendation: SiteLint should have explicit WCAG 1.2.8 reporting for prerecorded synchronized media that lacks a full media alternative, whether through a dedicated rule or a broader media rule with correct SC mapping.

Failing example 2: Prerecorded video-only media with no media alternative

This fails because the page presents prerecorded video-only media but provides no full alternative for time-based media.

SiteLint observation: SiteLint flags this genuinely failing scenario, but again reports it under WCAG 1.2.1 rather than WCAG 1.2.8.

Recommendation: SiteLint should distinguish 1.2.8 media-alternative requirements from 1.2.1 audio-only and video-only requirements. This scenario should only support a 1.2.8 finding if SiteLint is explicitly evaluating the absence of a full media alternative.

Current 1.2.8 coverage takeaway

Recommendation: SiteLint appears to apply the same broad media rule used on the 1.2.1, 1.2.2, 1.2.3, and 1.2.5 pages, and it again reports all four results here only as WCAG 1.2.1.

On this 1.2.8 page, that means SiteLint incorrectly flags both passing examples that provide full media alternatives, while the two genuinely failing examples are only caught through a mis-mapped 1.2.1 finding. SiteLint needs more precise, accurately mapped media coverage for 1.2.8 rather than reusing a generic “no track or audio source” heuristic.
