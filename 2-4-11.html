WCAG 2.4.11: Focus Not Obscured (Minimum)

View main list

Brief summary: When a user interface component receives keyboard focus, the component must not be entirely hidden by author-created content.

View the W3C Understanding page for WCAG 2.4.11 Focus Not Obscured (Minimum).

Observed SiteLint behavior

SiteLint uses focus-not-obscured-minimum as a manual-verification trigger for WCAG 2.4.11. It does not appear to determine whether the focused button in the overlay example is actually obscured.

The example link points to a real page so unrelated same-page-link findings do not obscure focus-obscuration behavior.

Recommendation

Keep focus-not-obscured-minimum as a manual prompt unless SiteLint can focus each component in rendered viewports and compare the focused component's bounding box with author-created overlapping content. Confirmed findings should identify the obscured focused element and the content covering it.
