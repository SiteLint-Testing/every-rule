Image inside of a link needs an alternative description

Rule ID: img-empty-alt-in-link

Jira ticket: PM-1097

Client URL: http://develop.sponsorship.qa.internal.test/#/

This page demonstrates six scenarios to test SiteLint's detection of images with empty alt attributes inside links. The goal is to isolate this rule without triggering false positives like “links should have meaningful, descriptive content.”

✅ 1. Valid – Image with descriptive alt inside link:

This passes because the image has an appropriate alt text.

✅ 2. Valid – Decorative image with empty alt outside link:

This is acceptable. Decorative images may have alt="" as long as they are not inside a link.

✅ 3. Valid – Link has an aria-label and image with empty alt:

Decorative image is allowed if the link has a meaningful aria-label. This should pass.

✅ 4. Valid – Link includes visible text and image with empty alt:

The image is decorative, and the link has meaningful text. SiteLint should not flag this.

This is visible text associated with the image
❌ 5. Invalid – Image inside a link with empty alt:

This is the classic violation: an image inside a link with alt="". SiteLint should flag this for img-empty-alt-in-link.
