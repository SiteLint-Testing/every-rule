WCAG 1.4.3: Contrast (Minimum)

View main list

Brief summary: Text and images of text should have sufficient contrast against adjacent background colors. Normal text generally requires at least 4.5:1 contrast, while large-scale text generally requires at least 3:1 contrast.

View the W3C Understanding page for WCAG 1.4.3 Contrast (Minimum).

These examples focus on machine-detectable text contrast cases. This page does not attempt to test contrast inside images of text, because that requires image analysis or human review.

Expected secondary contrast result when AAA is enabled

If the WCAG AAA filter is enabled, SiteLint may also report WCAG 1.4.6 Contrast (Enhanced) on this page. That is expected because any text that fails WCAG 1.4.3 AA contrast will also fail the stricter AAA contrast threshold. To isolate only WCAG 1.4.3 results, disable AAA in the SiteLint filter or treat 1.4.6 as an unavoidable secondary contrast finding on this page.

Observed missed example 5: Placeholder text has insufficient contrast

This is a meaningful 1.4.3 failure because placeholder text can provide instructions or examples and must meet the same contrast requirements when it is used to convey information. During testing, SiteLint did not report this placeholder contrast failure.

Observed result: SiteLint did not flag the low-contrast placeholder text. This should be considered a potential rule-expansion area if SiteLint intends to evaluate placeholder text contrast under WCAG 1.4.3.

Observed difficult example 6: Text over a gradient background may have insufficient contrast

This is a meaningful 1.4.3 risk because the contrast of the text depends on the rendered pixels behind each part of the text. A CSS-only contrast check may not reliably evaluate gradients, images, video backgrounds, or patterned backgrounds.

Registration closes Friday at 5 PM.

Observed result: This type of issue may be missed or evaluated inconsistently if SiteLint relies only on computed foreground and background colors. The actual contrast depends on the rendered background behind the text.

Recommendation: Consider rendered-background analysis for text over gradients, images, video, or patterned backgrounds. This likely requires screenshot or pixel-based testing rather than CSS-only evaluation.

Observed difficult example 7: Parent opacity reduces rendered text contrast

This is a meaningful 1.4.3 risk because the computed text color may appear sufficient, while the rendered text is visually lighter because opacity is applied to an ancestor element.

This text may appear lower contrast because its parent has opacity.

Observed result: A rule that reads only the computed color and background-color may overestimate contrast if it does not account for opacity, alpha colors, overlays, or compositing effects.

Recommendation: Account for opacity on the text node and ancestor elements when calculating contrast. If SiteLint cannot compute the final composited colors reliably, this should be reported as a manual-review candidate rather than a confirmed pass.

Observed missed example 8: SVG text has insufficient contrast

This is a meaningful 1.4.3 risk because SVG <text> can be visible, meaningful text content. HTML-only text contrast checks may miss SVG text entirely.

Observed result: SiteLint may miss this if the contrast rule only evaluates HTML text nodes and does not inspect visible SVG <text> content.

Recommendation: Evaluate SVG <text> contrast when SVG text is visible and meaningful. Complex SVG cases involving masks, filters, transforms, layering, or patterned backgrounds may still require rendered analysis or manual review.

Observed SiteLint coverage notes

SiteLint correctly reports several intended text contrast failures on this page. It also incorrectly reports the decorative low-contrast divider, even though the divider characters are marked aria-hidden="true" and are not needed to understand the content.

SiteLint did not report the low-contrast placeholder text example, even though placeholder text can communicate instructions or examples and should meet the applicable contrast threshold when it conveys information. Additional difficult or potentially missed cases include text over gradients or images, text affected by ancestor opacity, and visible SVG text.

Recommended SiteLint rule improvements

Recommendation: Update the contrast rule so decorative text marked aria-hidden="true" is not reported as a confirmed WCAG 1.4.3 failure unless the content is still visually meaningful. When the rule cannot determine whether low-contrast visible content is meaningful, consider reporting it as a manual-review item rather than a confirmed issue.

Recommendation: Add placeholder contrast support if SiteLint intends to evaluate all text-like content that can convey instructions or examples. Placeholder text should be evaluated when it communicates information to users.

Recommendation: Add rendered-background or pixel-sampling support for text over gradients, images, video, and patterned backgrounds. CSS-only background-color checks may not reflect the actual pixels behind the text.

Recommendation: Account for opacity, alpha colors, overlays, and ancestor compositing when calculating contrast. These can make text visually lower contrast than the computed foreground color suggests.

Recommendation: Add SVG <text> contrast support when SVG text is visible and meaningful. More complex SVG rendering cases may still need manual review or screenshot-based analysis.
