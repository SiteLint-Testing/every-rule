WCAG 1.4.3: Contrast (Minimum)

View main list

Brief summary: Text and images of text should have sufficient contrast against adjacent background colors. Normal text generally requires at least 4.5:1 contrast, while large-scale text generally requires at least 3:1 contrast.

View the W3C Understanding page for WCAG 1.4.3 Contrast (Minimum).

These examples focus on machine-detectable text contrast cases. This page does not attempt to test contrast inside images of text, because that requires image analysis or human review.

Failing examples

Failing example 1: Normal text has insufficient contrast

This fails because the normal-size text does not meet the 4.5:1 minimum contrast ratio against the white background.

This paragraph uses normal-size gray text with insufficient contrast against a white background.

Failing example 2: Large text has insufficient contrast

This fails because even though the text is large and bold, it does not meet the 3:1 minimum contrast ratio for large-scale text.

Large bold text still needs at least 3:1 contrast.

Failing example 3: Button text has insufficient contrast

This fails because the white button text has insufficient contrast against the light blue button background.

Failing example 4: Link text has insufficient contrast

This fails because the linked text has insufficient contrast against the white background.

Failing example 5: Placeholder text has insufficient contrast

This fails because placeholder text can provide instructions or examples and must meet the same contrast requirements when it is used to convey information.
