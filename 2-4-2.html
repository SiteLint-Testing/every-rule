WCAG 2.4.2: Page Titled

View main list

Brief summary: Web pages need titles that describe their topic or purpose so users can identify the page, especially when switching tabs, reviewing browser history, or navigating with assistive technology.

View the W3C Understanding page for WCAG 2.4.2 Page Titled.

What SiteLint is checking for

SiteLint's unclear-page-title rule appears to check the active document's <title> element for three specific conditions: missing title, empty or whitespace-only title, and a small set of known generic/default title strings.

This is useful partial WCAG 2.4.2 coverage. This page intentionally uses the active document title Untitled Document so SiteLint should report the current page under its unclear-page-title rule. Other title scenarios below are shown as code samples because a single page can only have one active title value at a time.

Scenarios SiteLint is designed to catch

Scenario 1: Missing document title

SiteLint should report a page when no title element exists in the document head. This example is shown as a code sample because this test page itself needs a valid active title.

<head></head>

Scenario 2: Empty or whitespace-only document title

SiteLint should report a page when the active title element is empty or contains only whitespace.

  • <title></title>
  • <title> </title>

Scenario 3: Known generic/default document title

SiteLint should report several exact generic/default title strings. Based on the current rule behavior, these include Untitled Document, No Title, Untitled Page, New Page 1, and Enter the title of your HTML document here.

  • <title>Untitled Document</title>
  • <title>No Title</title>
  • <title>Untitled Page</title>
  • <title>New Page 1</title>
  • <title>Enter the title of your HTML document here</title>

Scenario 4: Generic titles SiteLint may not catch

Some titles may still be too vague to satisfy WCAG 2.4.2, but SiteLint's current rule does not appear to evaluate title quality beyond missing, empty, whitespace-only, and a small set of exact default strings.

  • <title>Home</title>
  • <title>Page</title>
  • <title>Welcome</title>
  • <title>APAP</title>
  • <title>Dashboard</title>

Expected limitation: SiteLint may not report these titles because they are not part of the rule's known generic/default title list. Whether they fail WCAG 2.4.2 depends on the page context and whether the title actually describes the page topic or purpose.

Scenario 5: Duplicate titles across different pages

Duplicate titles across different pages can make it difficult for users to distinguish pages in tabs, browser history, search results, or assistive-technology page lists. A single-page scan usually cannot determine this because it does not have site-wide title context.

  • /about.html: <title>APAP</title>
  • /membership.html: <title>APAP</title>
  • /conference.html: <title>APAP</title>

Expected limitation: SiteLint's current single-page rule should not be expected to detect duplicate titles across a site unless the platform performs a crawl or compares titles across previously scanned pages.

Where the current rule is lacking

SiteLint's unclear-page-title rule provides useful partial WCAG 2.4.2 coverage, but it does not fully evaluate whether a page title describes the page's topic or purpose. It appears to rely on objective checks for missing, empty, whitespace-only, and several exact default title strings.

The current rule does not appear to evaluate vague but common titles such as Home, Page, Welcome, Dashboard, or short organization-only titles such as APAP. It also does not evaluate duplicate titles across a site, whether the title matches the visible page topic, or whether a single-page application updates the document title after route changes.

Recommended SiteLint treatment

Recommendation: Keep iframe title detection separate from WCAG 2.4.2 unless the rule is explicitly evaluating the title of the embedded document itself. Missing iframe title is primarily an iframe accessible-name issue and fits better under WCAG 4.1.2, so this page now avoids that unrelated finding.

Recommendation: To test WCAG 2.4.2 directly, create separate dedicated pages with active document-title failures, such as missing title, empty title, whitespace-only title, and each known generic/default title string that SiteLint checks. A single valid test page cannot simultaneously demonstrate active missing, empty, and generic document-title failures without changing the page's actual title element.

Recommendation: Consider expanding the rule to flag additional likely-generic titles such as Home, Page, Welcome, Dashboard, and very short organization-only titles when they do not appear to describe a specific page topic or purpose. These may need to be review candidates rather than confirmed failures because context matters.

Recommendation: Add site-wide or platform-level title analysis for duplicate titles across different URLs, and add single-page-application testing to verify that route changes update the active document title.
